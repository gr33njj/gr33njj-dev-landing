# Как вести блог

Админ-панели нет — мы используем "Git-based CMS". Это надежно, бесплатно и удобно для инженера.

## Как добавить новый пост:

1. Зайдите в папку `content/blog/`:
   ```bash
   cd /var/www/gr33njj.dev/html/content/blog
   ```

2. Создайте новый файл с расширением `.mdx`.
   Название файла должно быть в формате `slug-lang.mdx`.
   
   Пример для поста "My New Project":
   - Английская версия: `my-new-project-en.mdx`
   - Русская версия: `my-new-project-ru.mdx`

3. Вставьте в начало файла метаданные (Frontmatter):

   ```markdown
   ---
   title: "Заголовок вашего поста"
   date: "2024-11-28"
   excerpt: "Краткое описание, которое будет видно в списке постов."
   tags: ["tag1", "tag2", "tech"]
   lang: "ru"
   ---

   # Ваш заголовок

   Здесь пишите текст поста в формате Markdown.
   Можно использовать **жирный**, *курсив*, списки и код:

   \`\`\`python
   print("Hello World")
   \`\`\`
   ```

4. Сохраните файл.

5. Если вы работаете локально: сделайте `git commit` и `git push`.
   Если на сервере: просто сохраните файл.

6. **Важно:** Если вы добавляете посты прямо на сервере, нужно перезапустить сборку, чтобы Next.js увидел новые файлы (так как это статический генератор для блога):
   
   ```bash
   cd /var/www/gr33njj.dev/html
   npm run build
   pm2 restart gr33njj
   ```

   *Примечание: В dev-режиме (`npm run dev`) перезапуск не нужен, посты появятся сразу.*
